# Gettext Autotranslate Event

This event is a part of the [Zolinga Internationalization](:Zolinga Intl) module. It is triggered from `cli` origin (command line) by the `bin/zolinga gettext:autotranslate` command. It uses the AI translator service (`$api->translator`) to automatically translate untranslated gettext entries in `.po` files.

## How It Works

For each domain and each supported locale (except `en_US`):

1. Loads the `.po` file (or `.po.autotranslate` if a previous session was interrupted).
2. Finds all untranslated entries via `GettextPoFile::getUntranslatedEntries()`.
3. For each untranslated entry, sends the full PO entry string to `$api->translator->translate()` with instructions to fill in the `msgstr` key(s).
4. Saves progress after each translation to `{locale}.po.autotranslate`.
5. When all entries are translated, merges the autotranslate file back into the original `.po` file using `msgmerge --compendium` (existing translations in the `.po` take precedence).
6. Removes the `.autotranslate` file on successful merge.

## Usage

```bash
bin/zolinga gettext:autotranslate [--domains={DOMAINS}]
```

`--domains` accepts a comma-separated list of domain names (e.g. `--domains=mod1,mod2`).

When no `--domains` option is provided, the command autotranslates all domains.

## Resuming Interrupted Sessions

If the process is interrupted (e.g. network error, crash), the `.po.autotranslate` file preserves all completed translations. Running the command again will pick up where it left off.

## Parallel Execution

You can run multiple `bin/zolinga gettext:autotranslate` processes simultaneously. Each process acquires a per-domain-and-locale lock (`autotranslate:{domain}:{locale}`), so parallel execution is safe and only makes sense when you have more than one domain+locale combination to translate.

The AI translator backend has a `concurrency` setting in `config/global.json` (for example, under `ai.backends.translator.concurrency`) that controls how many simultaneous translation requests the service handles. As long as that limit supports it, running multiple CLI instances speeds up the overall workflow.

Example — translate in parallel:

```bash
# Terminal 1
bin/zolinga gettext:autotranslate --all

# Terminal 2
bin/zolinga gettext:autotranslate --all
```

## Requirements

- The `$api->translator` service must be available (provided by `zolinga-intl`).
- The domain must have a `messages.pot` file (generated by `gettext:extract`) for the final merge step.
- The `msgmerge` CLI tool must be installed.

## Example

```bash
# Autotranslate all domains
bin/zolinga gettext:autotranslate

# Autotranslate specific domains
bin/zolinga gettext:autotranslate --domains=ipdefender,zolinga-cms
```
